# 正二十面体球形行星地形设计方案

> 目标：以正二十面体（Icosahedron）为基础构建球形行星地形，地形高低采用**噪声叠加（fBm）**方式生成。
> 本方案与《四叉树 LOD 地形算法分析》一脉相承——把"正方形块 + 四叉树"换成"三角形面 + 三角形四叉树"，评价函数、T 型裂缝等核心思想直接复用。

---

## 总体架构

```
基础正20面体(12顶点/20三角面)
        │  归一化到球面
        ▼
每个三角面 = 一棵三角形四叉树(patch tree)的根
        │  按视点距离/误差 递归细分(1→4)
        ▼
叶子 patch → 生成网格顶点
        │  顶点位置 p(单位球面点)
        ▼
3D 噪声叠加(fBm) 求高度 h(p)
        │  沿法线抬升: P = p·(R + h)
        ▼
求法线 → 纹理/生物群系 → 提交GPU
```

---

## 一、为什么选正二十面体

| 方案 | 优点 | 缺点 |
|---|---|---|
| UV 球（经纬度） | 映射简单、纹理直接 | **两极挤压严重**、极点奇异、三角形大小极不均匀 |
| 立方体球（Cubesphere） | 6 个面各是规则四叉树网格，LOD 好做 | 6 棱 8 角有轻微畸变，映射需校正 |
| **正二十面体球（Icosphere）** | **顶点分布最均匀**、无极点奇异、三角形尺寸差异最小 | 三角形拓扑，邻接关系比四边形复杂 |

对行星地形，**均匀性**最重要——它决定噪声采样密度均匀、三角形不畸变。所以 Icosphere 是画质上限最高的选择，代价是三角形四叉树的邻接管理略麻烦。

> 上手建议：初次实现，立方体球更容易（6 个规则四叉树，直接套用四叉树地形算法）。本方案按指定的正二十面体来设计。

## 二、基础网格与细分

### 1. 基础正二十面体

12 顶点、20 三角面、30 边。顶点由黄金比 `φ = (1+√5)/2` 构造：

```
(0, ±1, ±φ), (±1, ±φ, 0), (±φ, 0, ±1)   共 12 个
```

全部归一化到单位球（长度 = 1）。这 20 个三角面，**每个都是一棵三角形四叉树的根节点**。

### 2. 三角形的 1→4 细分（核心）

取三条边的中点连线，分成 4 个小三角形（中间一个倒三角 + 三个角三角形）：

```
        v0                v0
        /\                /\
       /  \      →       /__\
      /    \            /\  /\
     /______\          /__\/__\
   v1        v2      v1  m12   v2
```

**关键：新顶点（边中点）要先归一化再乘半径**，否则细分出来仍是平面：

```
m01 = normalize(v0 + v1)   // 落在单位球面上，而非弦的中点
```

递归细分即得 Icosphere，也天然形成 **LOD 层级结构**（每细一层 = level+1）。

## 三、坐标系与球面映射（噪声无缝的关键）

每个地形顶点有两个身份：

1. **单位球面方向** `dir = normalize(vertexPosition)` —— 用来采样噪声。
2. **最终世界位置** `P = dir * (R + height)` —— 用来渲染。

**为什么必须用 3D 位置采样噪声？**

用 2D 高度图 + UV 贴球，必然在接缝和两极出现拉伸、错位、断裂（球面无法无缝展开成平面，数学必然）。

而**把单位球面上的 3D 坐标 `(x,y,z)` 直接喂给 3D 噪声函数**，噪声在整个球面上连续、无缝、无极点奇异：

```
height = fbm3D(dir)   // dir 是单位球面点，3D 噪声天然无缝环绕
```

## 四、噪声地形：fBm 叠加（核心）

### 1. 基础：分形布朗运动 (fBm) = 多个八度噪声叠加

推荐 3D **OpenSimplex2**（无专利、各向同性好）；Perlin / Simplex 亦可。

```
h = 0; amplitude = 1; frequency = baseFreq
for i in 0..octaves:
    h += amplitude * noise3D(dir * frequency)
    frequency *= lacunarity     // 典型 2.0，每层频率翻倍
    amplitude *= gain           // 典型 0.5(persistence)，每层振幅减半
```

- **octaves（八度数）**：4~8 层。越多细节越丰富，也越贵。
- **lacunarity（间隙度）**：2.0，控制"每层细节密度倍增"。
- **gain / persistence（持续度）**：0.5，控制"每层细节强度衰减"。
- **baseFreq（基础频率）**：控制最大地貌尺度（大陆大小）。

低频层决定大陆/大地貌，高频层叠加出山脊、丘陵、碎石纹理。

### 2. 进阶：更像真实行星

- **Ridged Multifractal（脊状分形）——造山脉**
  ```
  n = noise3D(p)
  n = 1 - abs(n)      // 取绝对值再翻转 → 尖锐山脊线
  n = n * n           // 平方增强锐度
  ```
  用于中高频层，得到连绵山脉。

- **Domain Warping（域扭曲）——造弯曲山谷/侵蚀感**
  ```
  warp   = fbm3D(dir + offset)
  height = fbm3D(dir + warpStrength * warp)
  ```

- **大陆 / 海洋 mask**：一层很低频噪声做"板块"，调制山脉振幅（大陆才长高山，海洋压平）。海平面即高度阈值 `seaLevel`。

- **高度重映射**：对最终高度做曲线（如 `pow(h, k)`），让平原更平、高山更陡，符合真实分布。

### 3. 高度到位移

```
elevation = h_normalized * maxHeight    // maxHeight 相对半径，如 R 的 0.5%~2%
P = dir * (R + elevation)
```

> `maxHeight` 别太大：地球最高峰 8.8km / 半径 6371km ≈ 0.14%。游戏可夸张到 1%~3% 增强观感。

### 4. 一致性要求（最易踩的坑）

同一球面点 `dir`，无论从哪个 patch、哪个 LOD 层访问，`h(dir)` 必须返回**完全相同的值**（纯函数、无状态、只依赖 `dir` 与固定 seed）。否则相邻 patch 接缝处高度对不上，产生裂缝。

## 五、LOD 方案（复用四叉树思想）

### 1. 评价函数（决定 patch 要不要细分）

```
splitMetric = patchEdgeLength / distance(camera, patchCenter)
if splitMetric > threshold AND level < maxLevel:
    细分成4个子patch
else:
    生成网格并渲染
```

也可用**屏幕空间误差 (screen-space error)**：把 patch 几何误差（简化前后高度差 `dh`）投影到屏幕，超过 N 像素才细分，比纯距离更准。

### 2. 遍历

每帧从 20 个根开始，按相机位置遍历四叉树（BFS 双队列或递归），只有相机附近的 patch 深度细分。

### 3. 剔除

- **视锥剔除**：视锥外 patch 跳过。
- **地平线剔除**：球体背面 patch 整块剔除——球形地形独有的巨大优化，能砍近一半的面。

## 六、T 型裂缝处理（球面版）

### 方案 A：边中点剔除（几何精确，无额外顶点）

控制相邻 patch 级别差 ≤ 1，生成三角形时，**如果某条边的邻居是更粗的 patch，就跳过这条边上的加密顶点**，与粗邻居对齐。与四叉树地形文章"忽略 4 号顶点"完全一样。

- 优点：无缝、无多余三角形。
- 缺点：三角形四叉树**邻接查询复杂**（维护每条边邻居指针，跨根三角面时邻接方向会翻转）。

### 方案 B：Skirt（裙边）——工程最省事

给每个 patch 沿边缘向球心方向"垂下"一圈裙边三角形，把级别不同产生的缝从下方挡住。

- 优点：**实现极简**，patch 完全独立生成，无需邻接关系。
- 缺点：极少量重叠三角形浪费；掠射角度偶尔露馅（调裙边深度）。

> 建议：**先用 Skirt 跑通**，画质/性能有极致要求再换边中点剔除。CDLOD、很多商业行星引擎都用 skirt。

## 七、法线与光照

fBm 无解析梯度，法线一般这样求：

- **有限差分**：在切平面取两个微小偏移点，采样高度后叉乘。
  ```
  采样 dir 附近的 P、P+du、P+dv → 法线 = normalize(cross(P+du - P, P+dv - P))
  ```
- **网格法线**：由三角形面法线累加到顶点（更简单，精度依赖网格密度）。
- **法线贴图**：高频细节烘焙进纹理，几何只到中频，省顶点。

## 八、精度问题（行星尺度绕不开）

行星半径动辄百万米，`float`（约 7 位有效数字）在远离原点时顶点抖动、Z-fighting。必须处理：

1. **Camera-relative rendering**：每帧平移世界，让相机永远在原点附近。
2. **CPU 侧用 double**：四叉树遍历、patch 中心、距离计算用双精度，最后转 float 提交。
3. **分块局部坐标**：每个 patch 顶点相对 patch 原点存储。
4. **对数深度缓冲**：解决地表到太空巨大深度范围的 Z 精度问题。

---

## 九、纹理与生物群系（可选扩展）

- **三平面映射 (Triplanar mapping)**：球面无天然 UV，用 xyz 三方向投影混合，避免拉伸。
- **按高度/坡度/纬度分生物群系**：海洋→沙滩→草地→岩石→雪。纬度用 `dir.y` 近似，坡度用法线与 `dir` 点积。
- 在 shader 里按 `height / slope / latitude` 实时混合。

## 十、生成流水线：CPU vs GPU

| | CPU 生成 | GPU 生成 (compute shader) |
|---|---|---|
| 灵活性 | 高，易调试 | 较低 |
| 速度 | 慢，适合分帧/异步 | 快，适合实时大量 patch |
| 推荐阶段 | 起步、原型 | 成熟后优化 |

**推荐路线**：
1. **CPU 单线程**跑通：细分 → 噪声 → 显示，验证正确性。
2. **多线程 + 异步生成**：patch 网格后台生成，主线程只调度渲染，对象池复用。
3. 极致性能：噪声与网格生成搬到 **compute shader**。

## 十一、核心数据结构（伪代码）

```cpp
struct PatchNode {
    Vec3d  v0, v1, v2;      // 三个角(单位球面方向, double)
    int    level;           // LOD 级别
    int    faceId;          // 属于哪个根三角面(0~19)
    PatchNode* children[4]; // 细分后的4个子patch
    AABB   bounds;          // 世界空间包围盒(视锥/地平线剔除用)
    Mesh*  mesh;            // 叶子才有(顶点/索引缓冲)
};

// 全局纯函数：唯一的高度来源
double heightAt(Vec3d dir) {           // dir 已归一化
    return fbm3D(dir, seed, octaves, lacunarity, gain);
}

Vec3d displace(Vec3d dir) {
    return dir * (RADIUS + heightAt(dir) * MAX_HEIGHT);
}
```

主循环：

```
for each of 20 root faces:
    traverse(rootPatch, camera)

traverse(patch, cam):
    if 视锥外 or 地平线背面: return          // 剔除
    if 需要细分(patch, cam) and level<max:
        确保 patch.children 已创建
        for c in children: traverse(c, cam)
    else:
        if patch.mesh == null:
            patch.mesh = buildMesh(patch)    // 细分成网格顶点, displace, 加skirt, 求法线
        render(patch.mesh)
```

## 十二、实施里程碑（建议顺序）

1. **静态 icosphere**：正20面体 + 固定层数细分 + 归一化，先看到光球。
2. **加噪声**：3D fBm displace，看到起伏行星（先不做 LOD，用中等固定细分）。
3. **调噪声**：加 ridged / domain warp / 大陆 mask / 海平面，把地貌调好看。
4. **加 LOD**：三角形四叉树 + 距离评价，能贴近看细节、拉远省性能。
5. **修裂缝**：先 skirt，跑顺。
6. **精度**：相机相对渲染 + 对数深度，从太空俯冲到地表不抖。
7. **法线/纹理/生物群系**：上光照和材质。
8. **优化**：异步生成、对象池，必要时 GPU 化。

---

## 精髓总结

- **正20面体保证顶点均匀** → 每个面挂一棵三角形四叉树做 LOD。
- **3D 噪声采样单位球面点** → 天然无缝、无极点奇异，沿法线抬升即得地形。
- **LOD 评价函数 + T 型裂缝处理直接复用四叉树地形那套思想**，几何从正方形换成三角形。
