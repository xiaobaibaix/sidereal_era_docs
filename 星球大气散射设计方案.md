# 星球大气散射设计方案（three_planet）

> 配套理论：`docs/大气散射技术分析.md`
> 目标：把当前 `three_planet/effects.js` 里的"假 fresnel 辉光"升级为**真实的瑞利 + 米氏单次散射**，支持从太空到地表自由飞行。
> 现有相机架构：主相机(OrbitControls) / 旁观相机(PointerLock, WASD) / 角色相机(walker)，画中画双视口 `renderViews()`。方案要**三种视角都成立**。

---

## 1. 现状与差距

`effects.js::createAtmosphere()` 现状：一个 `BackSide` 球壳（半径 = `radius*1.22`），freshel 边缘 `pow(0.72 - dot(N,viewZ), power)`，加法混合，颜色固定 `uColor=0x5a99ff`。

| 能力 | 现状 | 目标 |
|---|---|---|
| 地平线蓝带 | 固定色，无渐变 | 随视线/太阳角度物理渐变 |
| 日出日落红移 | ❌ | ✅ 斜射时红/橙（1/λ⁴ 效应）|
| 太空看行星光环 | 边缘一圈死蓝 | ✅ 蓝色光环 + 昼夜过渡 |
| 晨昏线（terminator）| ❌ | ✅ 白天侧亮、夜侧暗 |
| 太阳米氏光晕 | ❌ | ✅ 太阳附近白亮前向散射 |
| 地表受大气影响（aerial perspective）| ❌ | ✅ 远处地表泛蓝、被大气"洗淡" |

**技术选型**：采用 `大气散射技术分析.md` 的**路线 A（实时单次散射 raymarch）**。原因：
- 单个球形行星，几何简单；
- three.js 里没有现成 ComputeShader/LUT 预计算管线，raymarch 纯片元 shader 即可；
- 需求是"太空↔地表自由飞行" + 太阳方向可动，raymarch 天然支持任意视点、无需重算 LUT；
- 步数（N=16/M=8 量级）性能可接受。
- 路线 B（Bruneton LUT）留作后期性能/画质升级的可选项（见 §7 里程碑 M6）。

## 2. 渲染架构选择：全屏后处理 pass（推荐）

两种画大气的方式：

### 方案 2A：大气壳球（改造现有 mesh）—— 不推荐作为最终形态
在现有 `BackSide` 球里做 raymarch。问题：从**地表内部**看时相机在壳内，`BackSide` 剔除、深度关系都要特判；且拿不到"地表已经画好的颜色"做 aerial perspective。

### 方案 2B：全屏 raymarch pass（推荐）
先正常渲染场景（行星 + 海洋）到一张带**深度**的 render target，再用一个铺满屏幕的三角形/quad 跑大气 shader：
- 从相机位置沿每像素视线 raymarch；
- 用**场景深度**决定视线在哪里被地表挡住（积分区间的 far 端）；
- 大气色**叠加/混合**到已画好的场景色上 → 自动获得 aerial perspective（远处地表泛蓝）。

**这套后处理对三种相机天然通用**：谁在主画面就用谁的 view/projection 矩阵，`renderViews()` 每个视口各跑一次即可。

> 折中起步：里程碑 M1/M2 可以先用**天空盒式**的简化（只画天空、不读深度、不叠地表），快速看到从地表抬头的天空效果；M3 起再接入深度做太空视角和 aerial perspective。见 §7。

### three.js 落地方式
- 用 `THREE.WebGLRenderTarget`（`type: HalfFloatType`，需要 HDR 中间结果）存场景颜色 + 深度（`depthTexture: new THREE.DepthTexture()`）。
- 大气 pass 用一个 `THREE.Scene` + 正交相机 + 全屏 quad（或 `postprocessing` 的 `EffectComposer`；但我们自定义程度高，直接手写 quad 更可控）。
- 注意：现在 `renderViews()` 直接 `renderer.render(scene, cam)`。接入后处理后要改成：**render 到 RT → 大气 pass 到屏幕**，每个视口都走这条链。

## 3. 数学模型（落地版）

坐标单位：项目里行星 `radius=100`（世界单位），不是米。**保持无量纲**，用相对高度 `0=海平面, 1=大气顶` 或直接用世界坐标 + 标高比例。推荐做法：把标高 `H_R, H_M` 也按"行星半径比例"给（例如 `H_R = 0.085 * radius`），这样换半径时大气比例自动一致。

关键量（对应技术分析 §3~§6）：

```
R_ground = radius + seaLevel*maxHeight     // 地面球半径（跟海洋对齐）
R_atmo   = radius * ATMO_SCALE             // 大气顶半径, ATMO_SCALE≈1.15~1.25
H_R      = radius * 0.085                  // 瑞利标高比例
H_M      = radius * 0.012                  // 米氏标高比例（更贴地）
β_R      = vec3(5.8, 13.5, 33.1) * k_R     // 海平面瑞利系数(蓝>绿>红), k_R 调总强度
β_M      = vec3(21.0) * k_M                // 米氏系数(灰), 消光≈β_M*1.1
g        = 0.76                            // 米氏前向峰
sunDir   = normalize(sun.position)         // 来自 main.js 的平行光
sunColor, sunIntensity
```

相位函数：

```glsl
float phaseRayleigh(float mu) {            // mu = cos(视线,太阳)
    return 3.0/(16.0*PI) * (1.0 + mu*mu);
}
float phaseMie(float mu, float g) {
    float g2 = g*g;
    float num = (1.0 - g2) * (1.0 + mu*mu);
    float den = (2.0 + g2) * pow(1.0 + g2 - 2.0*g*mu, 1.5);
    return 3.0/(8.0*PI) * num / den;
}
```

密度（相对海平面高度的指数衰减）：

```glsl
vec2 densityAt(vec3 p) {                    // .x=瑞利, .y=米氏
    float h = length(p) - R_ground;
    return exp(-h / vec2(H_R, H_M));
}
```

## 4. Shader 算法（核心）

全屏 pass 片元着色器伪代码：

```glsl
// uniforms: camPos, invViewProj(重建视线), sunDir, sceneColor, sceneDepth,
//           R_ground, R_atmo, betaR, betaM, HR, HM, g, sunIntensity, N_VIEW, N_SUN

vec3 rd = reconstructRayDir(uv, invViewProj);   // 该像素世界空间视线方向
vec3 ro = camPos;

// 1) 与大气顶球求交，拿到视线在大气内的区间
vec2 atmo = raySphere(ro, rd, planetCenter, R_atmo);
if (atmo.y < 0.0) { out = sceneColor; return; }  // 视线没穿过大气

// 2) 用场景深度/地面求交裁剪 far 端（被地表挡住的地方停下）
float tGround = raySphere(ro, rd, planetCenter, R_ground).x;
float sceneT  = depthToDistance(sceneDepth, uv);  // 场景已画物体的距离
float tStart  = max(atmo.x, 0.0);
float tEnd    = min(atmo.y, min(tGround>0?tGround:INF, sceneT));

// 3) 视线积分（内散射）
float stepV = (tEnd - tStart) / N_VIEW;
vec3  sumR = 0, sumM = 0;
vec2  odView = 0;                                  // 视线侧累计光学深度(R,M)
for (i in N_VIEW) {
    vec3 p = ro + rd * (tStart + (i+0.5)*stepV);
    vec2 d = densityAt(p) * stepV;
    odView += d;

    // 4) 每个采样点向太阳做外散射光学深度
    vec2 sunHit = raySphere(p, sunDir, planetCenter, R_atmo);
    // 若 p→太阳被行星本体挡住 → 该点在阴影里, 跳过内散射(晨昏线/夜侧变暗)
    if (!rayHitsGround(p, sunDir)) {
        float stepS = sunHit.y / N_SUN;
        vec2 odSun = 0;
        for (j in N_SUN) {
            vec3 q = p + sunDir * ((j+0.5)*stepS);
            odSun += densityAt(q) * stepS;
        }
        // 双向透射率(瑞利+米氏消光)
        vec3 tau = betaR*(odView.x+odSun.x) + betaM*1.1*(odView.y+odSun.y);
        vec3 T   = exp(-tau);
        sumR += d.x * T;
        sumM += d.y * T;
    }
}

// 5) 合成
float mu = dot(rd, sunDir);
vec3 inscatter = sunIntensity * (
      sumR * betaR * phaseRayleigh(mu)
    + sumM * betaM * phaseMie(mu, g) );

// 6) aerial perspective: 地表颜色也被视线透射率衰减
vec3 Tview = exp(-(betaR*odView.x + betaM*1.1*odView.y));
vec3 color = sceneColor * Tview + inscatter;

out = tonemap(color);   // HDR → LDR，见 §5
```

关键点：
- **行星本体遮挡太阳** → 该采样点不接收阳光 → 自动产生**晨昏线**和夜侧变暗。
- **`odView` 累计**同时用于地表 aerial perspective（第 6 步）。
- 太空视角：相机在大气外，`tStart=atmo.x>0`，视线穿过大气壳边缘 → 蓝色光环自然出现。

## 5. 与项目集成

### 5.1 太阳方向
`main.js` 已有 `const sun = new THREE.DirectionalLight(...); sun.position.set(1,0.6,0.8)`。大气 uniform 直接 `uSunDir = sun.position.clone().normalize()`。海洋 `uSunDir` 也共用同一个源，保证水面高光和大气一致。**建议**：把太阳做成可动（GUI 或自动公转），日出日落才有戏。

### 5.2 半径联动
`layoutEffects()` 里已在 rebuild 时更新 ocean/atmosphere 缩放。新增：更新大气 uniform 的 `R_ground = radius + seaLevel*maxHeight`、`R_atmo = radius*ATMO_SCALE`、`H_R/H_M`（按半径比例）。

### 5.3 三相机 / 画中画
后处理是**逐相机**的：`renderViews()` 主视口用 `mainCam()`、小窗用 `insetCam()`，各自：`render 场景→RT` 然后 `大气pass→对应视口`。需要每个相机的 `invViewProjection` 和 near/far 用于深度重建。**注意**小窗和主窗分辨率不同，RT 尺寸要对上（或各自一张 RT）。

### 5.4 星空 / 背景
现在 `scene.background = 0x05070d` + 3000 星点。大气 pass 里若视线**没进大气**（`atmo.y<0`）直接透传 sceneColor（星空可见）；进大气但穿出（太空看边缘）则星空 + 光环叠加。夜侧天空应接近星空色。

### 5.5 HDR / Tonemap
散射结果是 HDR（太阳附近很亮）。建议开 `renderer.toneMapping = THREE.ACESFilmicToneMapping` + `toneMappingExposure`，或在大气 pass 末尾手动 tonemap。否则地平线/太阳会死白。

### 5.6 GUI
在"外观"folder 下把现有 `大气强度/大气颜色` 替换/扩展为：
- `大气开关`、`大气顶比例 ATMO_SCALE`、`太阳强度`
- `瑞利强度 k_R`、`米氏强度 k_M`、`米氏 g`
- `视线步数 N_VIEW`、`太阳步数 N_SUN`（性能/画质权衡）
- `曝光 exposure`
- （可选）太阳角度 / 自动公转速度

## 6. 性能与坑

| 坑 | 说明 / 对策 |
|---|---|
| 双重循环开销 | N×M 次密度采样。起步 N=16,M=8；低端设备降到 12/6。可对小窗用更低步数。|
| half-float 精度 | 世界坐标用 `radius=100` 量级即可，别用真实 6360km（float 精度会崩）。标高用比例。|
| 深度重建 | 需要线性化深度 + `invViewProj` 重建世界视线。三相机 near/far 不同，逐相机传。|
| 与 LOD 行星深度冲突 | 行星是分块异步生成，深度图必须在**行星渲染完**之后取；确保 pass 顺序正确。|
| 大气顶与近平面 | 太空视角相机离行星远（camera.far=200000），raymarch 只在 `[atmo.x, atmo.y]` 内，别从相机一路 march。|
| 米氏 g 符号 | `mu = dot(rd, sunDir)`；若前后向反了，检查 g 正负与 `1+g²-2g·mu` 约定（见技术分析 §5.2）。|
| 加法混合 vs 物理合成 | 现有 atmosphere 用 AdditiveBlending；新方案是**在 pass 里显式合成**（`scene*T + inscatter`），不再靠 blend。|
| 海洋与大气顺序 | 海洋是半透明球（renderOrder=1）。先画行星+海洋进 RT，再跑大气 pass，海洋色会正确参与 aerial perspective。|

## 7. 分里程碑实施步骤

> 沿用项目习惯：每个里程碑**先实现让你测试，满意后再提交**（可在新分支，如 `atmosphere-scattering`）。

- **M1 — 天空壳 raymarch（地表抬头看）**
  改造 `effects.js`：大气壳球内做瑞利单次散射 raymarch（先不接深度、不叠地表），太阳可动。产出：站在地表能看到蓝天、日落变红、太阳方位对。**验收**：角色模式下抬头，太阳附近亮、天顶蓝、地平线泛白/橙。

- **M2 — 米氏 + 相位 + 晨昏**
  加米氏散射 + HG 相位（太阳光晕）+ 行星遮挡太阳判定（夜侧变暗）。**验收**：太阳周围有白晕，昼夜有过渡。

- **M3 — 全屏后处理 + 深度 + aerial perspective**
  切到 §2B 全屏 pass 架构：场景→HalfFloat RT（带 DepthTexture）→大气 pass。接入地面/场景深度裁剪积分区间，地表颜色乘视线透射率。**验收**：从太空看行星有蓝色光环 + 昼夜面；远处山脉泛蓝变淡。三相机 + 画中画都正确。

- **M4 — HDR/Tonemap + GUI 调参**
  ACES tonemap + 曝光；GUI 暴露 k_R/k_M/g/步数/曝光/太阳。**验收**：不再死白，可实时调出地球/火星等不同大气观感。

- **M5 — 臭氧 + 抖动/降噪 + 性能**
  透射率加臭氧吸收项（青蓝/品红更真）；raymarch 步进加 dither 消除 banding；小窗降步数。**验收**：日落色更细腻，无明显条带，帧率达标。

- **M6（可选）— 预计算 LUT 升级**
  若性能吃紧或要多重散射，按技术分析 §7 路线 B 用 RT + 全屏 pass 预计算 (h, cos v, cos s) 3D LUT（three.js 无 ComputeShader，用多次 fragment pass 代替），运行时查表。**验收**：帧率显著提升、支持多重散射。

```js
const atmo = {
  enabled: true,
  atmoScale: 1.20,        // R_atmo = radius*1.20
  sunIntensity: 20.0,     // 配合 tonemap
  betaR: [5.8, 13.5, 33.1], // ×1e-? 归一到 radius 尺度, 用 kR 统调
  kR: 1.0,
  betaM: 21.0,
  kM: 1.0,
  mieG: 0.76,
  scaleHeightR: 0.085,    // ×radius
  scaleHeightM: 0.012,    // ×radius
  nView: 16,
  nSun: 8,
  exposure: 1.0,
  ozone: false,           // M5 再开
};
```

（`betaR/betaM` 的绝对值需在 radius=100 尺度下用 `kR/kM` 现场调到好看——真实值是按米/公里定的，直接用会偏。M1 落地时先手调出蓝天，再固化默认。）

## 9. 小结

- **要做的事**：把 `effects.js` 的假辉光换成单次散射 raymarch；分两阶段——先壳内天空（M1/M2），再全屏后处理接深度（M3+）。
- **为什么这样选**：单行星 + three.js 无 ComputeShader + 太空到地表自由飞行 → 实时 raymarch 最划算；LUT 留作后期。
- **难点**：全屏 pass 的深度重建、三相机/画中画各跑一遍、HDR tonemap、radius=100 尺度下的系数手调。
- **兼容性**：后处理逐相机通用，海洋先入 RT 再参与大气合成，不破坏现有 LOD/角色/旁观功能。
