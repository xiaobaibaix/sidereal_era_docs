# 真实地形:从噪声叠加到地质模拟的融合思路

面向对象:`docs/solar-system-modern.html`(纯噪声派)想往更真实地图演进。
参考对象:World Orogen(orogen.studio,模拟派)、procedural-planets-generator(C#)、Planet-Generator(Godot)。

> 已配套实装:本文 Tier 1 已经落到 `solar-system-modern.html`(见文末「实装记录」)。

---

## 0. 一句话结论

**噪声派和模拟派不是二选一。最真实的地形都是「噪声提供细节 + 结构层提供成因」的混合体。** 你不需要真去做板块模拟,只要往噪声上补几个「因果结构层」,真实感就能翻好几倍。

---

## 1. 把 Orogen 的思路抽象成 4 条「因果链」

Orogen 真实,是因为地形有**成因和方向性**,不是各向同性噪声:

1. **结构层(板块)**:先有板块分区 → 决定陆/洋,而不是「高度过阈值」。
2. **构造层(边界抬升)**:山脉沿板块边界生长(碰撞→山系,俯冲→海沟+海岸山脉)。**山有走向**。
3. **侵蚀层**:水力/热力/冰川改造地形——切河谷、塌碎石坡、刻峡湾。**地形有流向**。
4. **气候层**:纬度定温度、地形挡雨影 → 降水 → Köppen 分类 → 生物群系。**颜色符合气候**。

## 2. 为什么纯噪声看着「假」

FBM 本质是**各向同性、平稳、自相似**:每个尺度都像同一团「揉皱的纸」,没有大尺度组织原则。真实地形相反:

- 山脉是**线状**(沿边界),噪声给的是**斑块状**。
- 河谷**树状汇流、单向下坡**,噪声无流向。
- 侵蚀让高处尖、低处平缓沉积,噪声上下对称。
- 颜色该由气候决定,纯噪声只按高度分段。

要点不是「换更好的噪声」,而是**给噪声补上方向性和因果性**。

---

## 3. 三档融合方案(按性价比排序)

### 🟢 Tier 1:纯函数级增强(实时,直接改 FBM,收益最大)

保持「给方向 dir → 算高度」的纯函数结构,不破坏实时性。

**(1) Domain Warping(域扭曲)—— 单条最高 ROI**
用一层噪声扰动另一层的采样坐标,地形从「斑块」变成「被冲刷过的流线」:
```glsl
vec3 q = vec3(fbm(p), fbm(p + o1), fbm(p + o2));
float h = fbm(p + warpStrength * q);
```

**(2) Ridged multifractal(山脊噪声)—— 造有走向的山**
`(1 - |noise|)²` 造尖脊;高海拔用 ridged、低海拔平滑:
```glsl
float ridge(vec3 p){ float n = 1.0 - abs(noise(p)); return n*n; }
float h = base + smoothstep(0.3, 0.8, base) * ridgedFbm(p*2.0) * 0.5;
```

**(3) Worley 边界脊 —— 廉价「板块边界」**
Worley 的 **F2−F1** 天然在细胞边界形成脊线,视觉上就像板块碰撞带。撒的细胞种子 = 「板块」:
```glsl
vec2 w = worley(p * plateFreq);      // F1, F2
float boundary = w.y - w.x;          // 边界处→0
float belt = pow(1.0 - clamp(boundary,0,1), 4.0);  // 边界起脊
h += ridgedFbm(p*2.0) * belt * strength;
```
> 这是「不做模拟却拿到山脉沿边界结构」的关键。

**(4) 大陆掩膜 —— 陆海成片**
别用 `noise > seaLevel` 直接切。先用低频噪声/Worley 做大陆掩膜,再在陆地叠细节(即 Planet-Generator 的 `use_first_as_mask`):
```glsl
float continent = fbm(p * 0.5);
float land = smoothstep(seaLevel-0.05, seaLevel+0.05, continent);
h = continentBase + land * (mountains + detail);   // 海里不叠山
```

**(5) 气候配色 —— 感知真实感的最大来源**
把「按高度分段」换成 **温度 × 湿度 → 生物群系**:
```glsl
float latitude = abs(dir.y);
float temperature = (1.0 - altitude01) * (1.0 - latitude);  // 越高/越极越冷
float moisture   = humidityNoise * (1.0 - altitude01);
vec3 color = biomeLUT(temperature, moisture);               // 2D 查找/渐变
// 极地/高海拔 → 雪盖
```

> Tier 1 小结:**域扭曲 + ridged + Worley 边界 + 大陆掩膜 + 气候配色**,全部纯函数、实时、直接改 FBM 循环。做完就从「噪声球」跨到「看着像真的」。

### 🟡 Tier 2:烘焙一次 + 侵蚀(真实感质变,失去「实时无限」)

真正把「假/真」分开的是**水力侵蚀**——噪声再调也造不出真实树状水系和沉积。要做必须:

1. 地形**烘焙成高度图**(为避两极畸变,烘到 **cube 6 面**而非等距柱状)。
2. 在高度图上跑**液滴水力侵蚀**(Sebastian Lague 那套:撒雨滴、顺坡携带/沉积泥沙,迭代数万~数十万滴,GPU compute 或 CPU 预计算)。
3. 顺带算**流量累积 → 河流**、**雨影**(选风向,山背风面湿度衰减)。

代价:程序从「实时参数化」变「**生成→查看**」模式(点一次 Generate 跑一遍,像 Orogen)。这是**架构分叉点**。

### 🔴 Tier 3:真·板块模拟(Orogen 本体)

Voronoi 板块 + 运动矢量 + 边界分类(收敛/离散/转换)→ 初始高程 → 侵蚀 → 气候 → Köppen。大工程,离线生成。除非做世界观级地图工具,否则 Tier 1 + 部分 Tier 2 已够「更真实的地图」。

---

## 4. 关键架构决策:你要哪种「真实」

| 目标 | 走哪档 | 代价 |
|------|--------|------|
| 实时可交互、随手调参(现在的 HTML) | **Tier 1** | 保持实时,真实感有限但性价比最高 |
| 真实水系/侵蚀/气候地图,能接受「生成一次再看」 | **Tier 1 + Tier 2** | 放弃实时无限,改 bake 模式 |
| 世界观/地图导出工具 | **Tier 3** | 大工程,离线 |

`solar-system-modern.html` 是「实时全量重建」架构,**Tier 1 完全兼容**。一旦要 Tier 2 侵蚀,就得引入「烘焙到纹理 + 后处理」,这是最大一步跨越。

---

## 5. 建议路线

1. **先做 Tier 1**(已实装):域扭曲 + ridged + Worley 边界 + 大陆掩膜 + 温湿度配色。收益/成本比最高,不动架构。
2. 看效果再决定要不要上 **Tier 2 水力侵蚀**——「像真的」到「是真的」的分水岭,但要接受 bake 模式。
3. Tier 3 除非做工具,否则不建议。

## 6. 实装记录(Tier 1 已落到 solar-system-modern.html)

在 `NoiseGenerator` 里新增:
- `ridgedFbm(x,y,z,octaves)`:带 octave 权重的山脊多重分形。
- `worley(x,y,z)`:3D Worley,返回 `{f1, f2}`(最近/次近特征点距离)。
- `_rand3(i,j,k,salt)`:每个细胞的确定性伪随机(供 Worley 用)。

新增地形/配色函数:
- `getTerrain(nx,ny,nz)`:域扭曲 + 大陆掩膜 + 中尺度 relief + Worley 边界带(`PLATE_FREQ`)+ ridged 山脉,输出 `0..1` 高度。
- `BIOME` 调色板 + `biomeColor(temperature, humidity)` + `lerp3`。

`createPlanet` 顶点循环改为:
- 由顶点算单位方向 `nx,ny,nz` → `getTerrain` 得高度 → 径向位移(沿用 `params.strength`)。
- 配色改为 **温度(纬度+海拔)× 湿度(噪声,随海拔变干)→ biome**,高海拔/寒冷叠 rock/snow;海面保留深浅水渐变;海岸线沙滩。
- **`gas`/`ice` 星球跳过 biome**,保留 `surfaceColor` 着色,避免气态巨行星被染成绿色。

参数复用:`scale`=频率,`strength`=起伏幅度,`oceanLevel`=海平面(语义不变,原 UI 滑块继续有效)。

注意事项 / 局限:
- 纯 CPU,`detail=150` 时每次重建约数百 ms(每个陆地顶点约 26 层噪声 + 27 个 Worley 细胞),属可接受的一次性成本;拖动滑块的 `change` 才触发。
- Worley 用整数哈希伪随机,非严格蓝噪声分布,但作为「板块边界」足够。
- 这仍是 Tier 1(结构增强),**没有真实水力侵蚀/河流/雨影**——那属于 Tier 2,需要烘焙高度图。

## 7. 一页脉络:三种流派

| | 噪声派 | 瓦片派(Oskar) | 模拟派(Orogen) |
|---|---|---|---|
| 地形来源 | 噪声高度场 | 预制模块拼贴 | 地质过程模拟 |
| 山脉成因 | 噪声凑 | 手摆 | 板块碰撞算出(有走向) |
| 气候/生物群系 | 无/近似 | 无 | 完整气候 + Köppen |
| 主产物 | 实时 3D 星球 | 可玩星球 | 可导出 2D 地图集 |
| 时间维度 | 无 | 无 | 有(漂移/侵蚀/火山拖尾) |
| 融合建议 | **加因果层升级** | — | 借其结构思路 |

核心配方:**噪声负责细节,结构层负责可信度**。用 Worley 边界当板块、域扭曲+ridged 当侵蚀观感、温湿度当气候配色;要更真再上真实水力侵蚀(需烘焙)。
