## 0. 文章定位与作者取舍

原文是一篇"**原理推导 + Unity 实现**"的长文，结构分两大块：

1. **原理篇**：从零推导瑞利散射、米氏散射，包括前置的麦克斯韦方程组、伴随勒让德多项式、球谐函数、球面贝塞尔函数等。
2. **实现篇**：用 Unity ComputeShader + LUT（查找表）做预计算大气散射；另外附带一段"上世纪 90 年代不用 LUT 的实时 raymarching 做法"（源自 GPU Gems2 第 16 章）。

作者自述**删掉了一半内容**（全动态实时天空、数值积分方式对比），所以本文缺"实时天空"和"积分格式对比"两块。对我们最有价值的是：**单次散射的积分模型**、**瑞利/米氏系数与相位函数**、以及**两条工程实现路线（实时 raymarch vs 预计算 LUT）**。

米氏散射的严格解（Mie 1908，解麦克斯韦方程组得到球形粒子弹性散射的精确解）推导冗长，**且在实时渲染里根本用不上**——最终工程都用 Henyey-Greenstein（HG）近似相位函数代替。所以本文对严格 Mie 推导只做结论性总结。

## 1. 物理图景：大气散射是"内散 + 外散"的积分问题

普通着色只描述**物体表面**的光照；大气散射描述的是**一束光在介质里反复偏转**的过程，因此 SDF 之类的表面技术不适用。

把散射按效果分两类：

- **外散射（out-scattering）**：光沿原路径前进时，被粒子偏转"散出去"，导致原方向光强**衰减**。
- **内散射（in-scattering）**：别的方向的光被偏转到**当前视线方向**，给视线"加光"。天空之所以是亮的、能看见光晕，就是内散射。

### 关键的一张图（切入点）

设定：

- 相机在 **A** 点，视线方向 A→B。
- 太阳光是唯一光源，因为地日距离极远，**近似为平行光**（方向处处相同）。
- 视线上任取一点 **P**：太阳光先从大气层外的 **C** 点穿到 P（路径 C→P 有衰减），在 P 处发生一次散射转向相机，P→A 路径上又继续衰减。
- 真实情况：视线 A→B 上有**无数个** P，A 看到的是所有 P 点内散射贡献的**积分求和**（数值积分）。

物理上 C→P 段和 P→A 段是同一种衰减过程，可以用同一套透射率来描述。

## 2. 透射率与光学深度（衰减的核心）

### 透射率 Transmittance

定义两点间光强的比值。光穿过大气会被散射衰减，衰减是**指数**的（Beer-Lambert 定律的物理直觉：每一小段都按当前密度散掉一个比例 β，无数次连乘 → 指数函数）：

```
T(C, P) = exp( -∫ β(h) ds )        沿 C→P 路径积分
```

- β 是**散射/消光系数**，取决于大气密度，随高度变化。
- 积分号里那部分（沿路径把大气密度累加起来）就是所谓的**光学深度 D(cp)**。

### 光学深度 Optical Depth

```
D(path) = ∫ ρ(h) ds
```

- ρ(h) 是**相对密度**（相对海平面），海平面为 1，随高度指数下降。
- 这是 shader 里**真正要用循环去算的量**；其余的 β、4π、波长项等都是**常数系数**，算一次即可。

> 工程要点：把透射率里的常数系数提出来，循环里只累加密度得到光学深度，最后再乘系数、取 exp。这是整个实现的性能关键。

## 3. 单次散射的完整积分方程

把上面拼起来，A 点某视线方向收到的单次散射光强：

```
L(A, view) = ∫[A→B]  T(C, P) · L_sun · β(h_P) · Phase(θ) · T(P, A)  ds
              └ P 沿视线积分 ┘
```

逐项含义：

| 项 | 含义 |
|---|---|
| `L_sun` | 大气层外的太阳光强（宇宙段视为真空，不衰减）|
| `T(C, P)` | 太阳光从大气外 C 到散射点 P 的透射率（外散射衰减）|
| `β(h_P)` | P 点高度的散射系数（密度越大散射越强）|
| `Phase(θ)` | 相位函数：入射与出射夹角 θ 上分配多少光（瑞利/米氏不同）|
| `T(P, A)` | 散射后从 P 回到相机 A 的透射率 |
| `ds` | 步长，数值积分的微元权重 |

**双重积分结构**：外层沿视线 A→B 采样 P；对每个 P，内层还要沿 **P→太阳方向** 采样算 `T(C,P)`（外散射光学深度）。这就是 raymarch 的"两层循环"来源。

## 4. 瑞利散射（Rayleigh）——天空为什么是蓝的

**适用条件**：粒子半径 ≪ 光波长（如空气分子 vs 可见光）。

### 4.1 散射系数（与波长四次方成反比）

```
β_R(λ, h) = ( 8π³ (n²-1)² / (3 N λ⁴) ) · ρ(h)
```

- `λ` 波长，`n` 折射率，`N` 海平面分子数密度，`ρ(h)` 相对密度。
- **∝ 1/λ⁴**：蓝紫光（短波）散射远强于红光 → 天空蓝、夕阳红（阳光斜射穿过更厚大气，蓝光早被散掉，剩红光）。
- 天空不是紫色，是因为人眼对蓝的敏感度高于紫。
- 海平面瑞利系数（RGB 三通道）是常数，通常直接给数值。

### 4.2 相位函数（角度依赖）

瑞利相位函数（描述散射角 θ 的分布）：

```
Phase_R(θ) = (3 / 16π) · (1 + cos²θ)
```

前向和后向大致对称，形状温和。

### 4.3 密度比（高度衰减）

```
ρ(h) = exp( -h / H_R )
```

- **H_R ≈ 8500 m**（原文称"刻高高度 / 标高 scale height"）。
- 瑞利散射主要发生在大气前 ~60 km（对流层），温度线性下降、压力指数下降，指数模型是很好的近似。

## 5. 米氏散射（Mie）——光晕、丁达尔效应

**适用条件**：粒子半径 ≈ 或 > 光波长（气溶胶、灰尘、水滴、烟雾）。

- 大部分光沿**前向**散射（强烈的前向峰），形成太阳周围的白色光晕、体积光。
- **与波长几乎无关**（所以雾/霾是白/灰的，不像天空分颜色）。
- 严格解由 Mie（1908）解麦克斯韦方程组得到（球谐函数 + 伴随勒让德多项式 + 球面贝塞尔函数展开），**精确但太贵，实时渲染不用**。

### 5.1 系数

工程上直接用便宜的经验式（海平面常数系数 × 密度比），米氏标高 **H_M ≈ 1200 m**（比瑞利低很多，气溶胶集中在近地）。

消光系数注意：分子只散射时 `β_e = β`；气溶胶**既散射又吸收**，`β_e_M = β_M + β_a_M`。

### 5.2 相位函数：Henyey-Greenstein（HG）近似

原始 HG（1941）：

```
Phase_HG(θ) = (1/4π) · (1 - g²) / (1 + g² - 2g·cosθ)^(3/2)
```

改进的**双 HG 单参数版**（1992，能同时体现前向峰）——原文实际采用的形式：

```
Phase(θ) = (3/8π) · [ (1-g²)(1+cos²θ) ] / [ (2+g²)(1 + g² - 2g·cosθ)^(3/2) ]
```

**g（非对称因子 / 首选散射方向）** 是关键：

- `g = 0` → 退化成各向同性，可用来**近似瑞利**。
- `g` 取 **-0.75 ~ -0.999**（原文的符号约定下负值代表强前向）→ 近似米氏前向峰。
- **切忌取 ±1**，分母/分子会变 0，散射消失。

> 注意符号约定坑：`cosθ = dot(-lightDir, viewDir)`。不同文献 g 的正负约定相反，移植时要看清 `1 + g² - 2g·cosθ` 里 cosθ 的定义，否则前后向会反。原文代码里调用 `Equation_mie(-0.78, cos, cosSquare)`。

## 6. 臭氧层（只吸收、不散射）

真实大气还有臭氧层，它**不散射光，只吸收光**（尤其影响天空颜色的青蓝过渡、日落的品红）。

处理方式：**只在透射率（消光）里加一项臭氧吸收**，不进相位/内散射。

```
ρ_O(h) ≈ 6e-7 · ρ_R(h)     臭氧密度（原文给的近似）
β_e = β_R·ρ_R + β_M·ρ_M + β_O·ρ_O
```

臭氧消光系数 β_O 有实测表可查。原文提到这是 2008 年 Bruneton 论文里**没做**、后人补上的。

## 7. 两条工程实现路线

### 路线 A：实时 raymarching（不用 LUT，GPU Gems2 第 16 章）

思路：直接在片元着色器里做上面第 3 节的双重积分。

1. **球体求交**：视线与"大气层球壳"求交，确定积分区间 `[near, far]`（进入大气→打到地面或穿出大气）。

   ```glsl
   float2 RaySphereIntersection(ro, rd, center, R) {
       ro -= center;
       float a = dot(rd, rd);
       float b = 2.0 * dot(ro, rd);
       float c = dot(ro, ro) - R*R;
       float d = b*b - 4.0*a*c;
       if (d < 0.0) return float2(-1);           // 未命中
       d = sqrt(d);
       return float2(-b - d, -b + d) / (2.0*a);  // 两个交点参数
   }
   ```

2. **密度函数**（指数衰减）：

   ```glsl
   float Density(vec3 p, float H) {
       return exp( -max(length(p) - r_inner, 0.0) / H );  // 相对海平面高度
   }
   ```

3. **外散射光学深度**（P→太阳方向累加密度）：

   ```glsl
   float OutScatter(pSource, pTarget, H) {
       float sum = 0.0;
       vec3 step = (pTarget - pSource) / OUT_SCATTER_COUNT;
       vec3 p = pSource;
       for (i in OUT_SCATTER_COUNT) { sum += Density(p, H); p += step; }
       return sum * length(step);
   }
   ```

4. **内散射主循环**（视线 A→B 采样 N 点，每点再向太阳做外散射 M 步）：

   ```
   for each sample point on view ray:
       d_view += Density(P) * stepLen                  # 累加视线侧光学深度
       hit = RaySphereIntersection(P, sunDir, atmoTop) # P 到大气顶
       d_sun = OutScatter(P, atmoEdge, H)              # 太阳侧外散射
       T = exp( -(d_view + d_sun) * β )                # 双向透射率
       sum_ray += Density_R(P) * T
       sum_mie += Density_M(P) * T
   result = sum_ray * β_R * Phase_R(θ) + sum_mie * β_M * Phase_M(g, θ)
   ```

- 复杂度约 **N × M** 次密度采样（原文常用 N=16, M=8 量级）。
- 优点：**代码不到 200 行、无需预计算、天然支持从太空到地表任意视点**。缺点：per-pixel 双重循环，步数一多就贵。
- 相位函数强度调参：原文里瑞利相位直接用 `(0.1875/π)(1+cos²)` 之类的经验缩放以获得合适亮度。

### 路线 B：预计算大气散射（Bruneton & Neyret 2008，LUT）

思路：光传输方程**与视点/视线/太阳方向都有关**，直接查表要 9 维（视点 xyz、视线 xyz、太阳 xyz），纹理爆炸。通过三条假设降维：

1. 地球是完美球体；
2. 大气密度**只随高度变化**，与经纬度无关；
3. 太阳光平行（去掉太阳方位角）。

降到 **3 个自由度 → 3D LUT**：

- **高度 h**
- **视线-天顶夹角 cos(v)**
- **太阳-天顶夹角 cos(s)**

（太阳-视线方位角被略去，代价是行星自阴影会略糊，靠把相位函数推迟到片元里评估来补。）

**参数化重映射**（Bruneton-Neyret 的精度技巧）：大气密度随高度指数下降，天空颜色差异在**低空**最大，所以三个参数到 [0,1] 的映射都**在低空/近地平线处加密采样**（非线性重映射），而不是均匀线性映射，用同样分辨率的纹理换更高精度。

**多重散射**：定义"k 阶聚焦函数"——恰好经历 k 次散射到达某点的光量。一阶 = 单次散射在所有方向积分并乘相位；高阶递推累加。最终把单次 + 各高阶结果相加，得到带多重散射的天空（2008 年后才有完整方法）。

LUT 生成用 Unity ComputeShader（`RWTexture2D` / `numthreads` / `SV_DispatchThreadID`，3D 体纹理 `RenderTextureFormat.ARGBHalf`），几秒算完，运行时**常数时间查表**，无采样。

- 优点：运行时极快、支持多重散射、画质高。
- 缺点：预计算复杂、需要 ComputeShader、LUT 内存、动态大气参数需重算 LUT。

### 补充：LUT 是什么

Look-Up Table，颜色/结果查找表，"用存储换计算"。3D LUT 用三个变量索引一个立方体，平面化展开成 2D 图存储（.cube/.3dl）。大气散射的 LUT 就是 (h, cos v, cos s) → 散射色。

## 8. 关键公式 / 常量速查

| 量 | 公式 / 值 |
|---|---|
| 透射率 | `T = exp(-∫β·ρ(h)ds)` |
| 光学深度 | `D = ∫ρ(h)ds`（循环累加密度） |
| 瑞利密度 | `ρ_R(h)=exp(-h/H_R)`, **H_R≈8500 m** |
| 米氏密度 | `ρ_M(h)=exp(-h/H_M)`, **H_M≈1200 m** |
| 瑞利系数 | `∝ 1/λ⁴`（蓝>绿>红），海平面给常数 RGB |
| 瑞利相位 | `(3/16π)(1+cos²θ)` |
| 米氏相位 | 双 HG，`g≈0.76`（或原文约定 -0.78） |
| 米氏特性 | 与波长无关、强前向峰、既散射又吸收 |
| 臭氧 | 只吸收，`ρ_O≈6e-7·ρ_R`，进消光不进相位 |
| 单次散射 | `∫ T(C,P)·L_sun·β·Phase·T(P,A) ds` |

地球类比常用数值（业界常见，供实现参考）：

```
行星半径 R_ground ≈ 6360 km,  大气顶 R_atmo ≈ 6420 km（厚约 60 km）
β_R (海平面) ≈ (5.8, 13.5, 33.1) × 10⁻⁶  m⁻¹   # R,G,B —— 蓝远大于红
β_M (海平面) ≈ 21 × 10⁻⁶ m⁻¹（散射），消光 ≈ 1.1×β_M
H_R = 8000~8500 m,  H_M = 1200 m,  g = 0.76
```

## 9. 对 three_planet 项目的启示

我们现在的 `effects.js::createAtmosphere()` 只是一个**背面球壳 + fresnel 边缘光 + 加法混合**的"假辉光"，好处是极便宜，但：

- 颜色是**固定的**（一个 uColor），不随太阳角度变（没有日出日落红移、没有地平线蓝带）。
- 没有**外散射衰减**——从太空看不到行星边缘真实的蓝色光环渐变，也不会给地表染色。
- 没有**米氏光晕**（太阳附近的白亮）。
- 没有**晨昏线**（day/night terminator）。

**结论**：适合升级到**路线 A（实时单次散射 raymarch）**——因为我们是单个球形行星、three.js 里没有现成 ComputeShader 管线、且需求是"从太空到地表自由飞行"，raymarch 天然满足；步数可控性能可接受。路线 B（LUT）作为后期画质/性能进一步优化的可选项。

具体如何落地到 `effects.js` / 新增全屏 pass、shader 结构、与主相机/旁观/角色/海洋的集成、分里程碑步骤和参数默认值，见配套文档：**`docs/星球大气散射设计方案.md`**。
